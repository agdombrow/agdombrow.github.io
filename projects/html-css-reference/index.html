Tables

  • structured set of data made up of rows and columns (tabular data)
  • allows you to quickly and easily look up values that indicate some kind of connection between different types of data
  • does it makes sense in a spreadsheet?
Tag Name Description
<table> Table The wrapper element for all HTML tables.
<thead> Table Head The set of rows defining the column headers in a table. Semantically indicate that rows contain no data but provide titles for the corresponding columns of a table. If included, no <tr> may be a direct child of <table>
<tbody> Table Body The set of rows containing actual table data.
<tfoot> Table Foot The set of rows defining the footer in a table. Semantically indicates rows contain no data but provide ancillary information. Must go after <tbody>.
<tr> Table Row The table row container.
<th> Tabular Header Individual cells of a table. Contains title for data.
<td> Tabular Data Individual cells of a table. Contains data.
<caption> Caption Caption describing the associated table. (Similar to the relationship between figcaption and figure).
<col> Column A column (a no-content element).
<colgroup> Column Group A groupd of columns.
Table Tags
Attribute Applicable To Description
colspan <th>, <td> Defines how many columns (horizontal) a <td> element should span. Accepts any positive integer.
rowspan <th>, <td> Defines how many rows (vertical) a <td> element should span. Accepts any positive integer.
span <col> Makes a column apply to two or more columns.
sortable <table> Indicates table should allow sorting. (Removed from spec for lack of implementation?)
headers <td> Space-separated string corresponding to the IDs of the <th> elements relevsnt to the data.
scope <th>, <td> Defines how many rows (vertical) a <td> element should span. Accepts any positive integer.

Best Practices

  • Use table-layout: fixed to create a more predictable table layout that allows you to easily set column widths by setting width on their headings (<th>).
  • Use border-collapse: collapse to make table elements borders collapse into each other (2px gap by default), producing a neater and easier to control look.

Reset/Normalize/Code Snippets Section?

    /* base reset/normalize */
    table {
      border-collapse: collapse;
      width: 100%;
    }

    th, td {
      padding: 0.25rem;
      text-align: left;
      border: 1px solid silver;
    }
    
    /* optional zebra-striping (add :nth-child(even) for further customization) */
    tbody tr:nth-child(odd) {
      background: silver;
    }
    
    /* optional highlighting on hover */
    /* 1. highlight individual cells */
    td:hover { /* th:hover also if you wish */
      background: whitesmoke;
    }

    /* 2. highlight rows */
    /* 2a. works as long so long as no declaration for td/th background... */
    tbody tr:hover {   
      background: whitesmoke;
    }

    /* 2b. ...otherwise, use this */
    tr:hover td, tr:hover th { 
      background: whitesmoke;
    }
